1 # Command Reference
2 
3 `playd` implements the BAPS3 Internal API (specifically the `End`, `FileLoad`,
4 `PlayStop`, `Seek`, and `TimeReport` features), as described in the [BAPS3
5 specification][]. The specification provides more formal definitions of the
6 commands supported by `playd`.
7 
8 ## Sending Commands Manually
9 
10 To talk to `playd`, open a raw TCP/IP session on the host and port on which
11 `playd` was started. For example, using [netcat][]:
12 
13  nc ${ADDRESS} ${PORT}
14 
15 On Windows, use [PuTTY][] in _raw mode_ with an _Implicit CR for every LF_.
16 
17 ## Initial Responses
18 
19 When connecting to `playd`, it will send a few initial responses to inform
20 the client of its current state. This will look something like:
21 
22  OHAI playd
23  FEATURES End FileLoad PlayStop Seek TimeReport
24  TIME 0
25  STATE Ejected
26 
27 This tells the client:
28 
29 * The name of the server program (`playd`);
30 * The BAPS3 features implemented by `playd`;
31 * The current position, in microseconds, into the current song (in this example,
32  it is `0` because there is no song)
33 * The current state (`Ejected`, which means no song is loaded).
34 
35 ## Command Format
36 
37 `playd` uses _shell-style_ commands:
38 
39 * Each command is a linefeed-delimited _line_ of whitespace-delimited _words_;
40 * Whitespace and quotes can be _escaped_ by using backslash;
41 * Words can be _quoted_ to avoid needing to backslash-escape large amounts of
42  whitespace: either using _double quotes_, which allows backslash escaping, or
43  _single quotes_, which doesn't;
44 * The first word represents the _command_, and each subsequent word is an
45  _argument_ to that command.
46 
47 ## Requests
48 
49 These are the requests that clients can send to `playd`. Request commands are
50 always in _lowercase_.
51 
52 ### quit
53 
54 Terminates `playd`.
55 
56 ### load _file_
57 
58 Loads _file_, which is an _absolute_ path to an audio file.
59 
60 ### eject
61 
62 Unloads the current file, stopping it if it is currently playing.
63 
64 ### play
65 
66 Starts playing the currently loaded file.
67 
68 ### stop
69 
70 Stops playing the currently loaded file. This does not alter the position;
71 use `seek 0` after `stop` to rewind the file.
72 
73 ### seek _position_
74 
75 Seeks to _position_, which may either be an integer (interpreted as microseconds
76 since the beginning of the file), or an integer followed by a suffix denoting
77 the unit. Units supported include `s` (seconds), `m` (minutes), `h` (hours),
78 `ms` (milliseconds), and `us` (microseconds).
79 
80 ## Responses
81 
82 These are the responses sent to clients by `playd`. Response commands are
83 always in _uppercase_.
84 
85 ### OHAI _name_
86 
87 Identifies the server program.
88 
89 ### FEATURES _feat1_ _feat2..._
90 
91 Lists the BAPS3 feature flags supported by `playd`.
92 
93 ### END
94 
95 Marks the end of a file. This is sent if a file finishes playing of its own
96 accord.
97 
98 ### TIME _position_
99 
100 Announces the current position in the file, in microseconds.
101 
102 ### STATE _state_
103 
104 Announces a state change. The _state_ will be one of:
105 
106 * `Playing` - the current file is now being played;
107 * `Stopped` - the current file is now stopped;
108 * `Ejected` - the current file has just been ejected;
109 * `Quitting` - `playd` is now quitting.
110 
111 ### FILE _file_
112 
113 Announces that _file_ has just been loaded.
114 
115 ### OKAY _command_
116 
117 Announces that a valid command has just been received and processed.
118 
119 ### WHAT _error_
120 
121 Announces that a bad command was just received.
