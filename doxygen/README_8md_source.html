1 # URY playd
2 
3 URY playd (or just `playd`; short for _player daemon_) is a C++ audio
4 player developed by [University Radio York]. It's designed to be
5 minimal, hackable, and composable into bigger systems (such as our
6 ongoing BAPS3 project to build a new radio playout system).
7 
8 
9 ## Usage
10 
11 `playd DEVICE-ID [ADDRESS] [PORT]`
12 
13 * Invoking `playd` with no arguments lists the various device IDs
14  available to it.
15 * Full protocol information is available on the GitHub wiki.
16 * On POSIX systems, see the enclosed man page.
17 
18 `playd` understands the following commands via its TCP/IP interface:
19 
20 * `load "/full/path/to/file"` — Loads /full/path/to/file for playback;
21 * `eject` — Unloads the current file;
22 * `play` — Starts playback;
23 * `stop` — Stops (pauses) playback;
24 * `seek 1000` — Seeks to 1,000 microseconds after the start of the current file.
25 * `quit` — Closes `playd`.
26 
27 ### Sending commands manually
28 
29 To connect directly to `playd` and issue commands to it, you can use
30 [netcat]:
31 
32  # If you specified [ADDRESS] or [PORT], replace localhost and 1350 respectively.
33  $ nc localhost 1350
34 
35 On Windows, using [PuTTY] in _raw mode_ (__not__ Telnet mode) with
36 _Implicit CR in every LF_ switched on in the _Terminal_ options should
37 work.
38 
39 __Do _not_ use a Telnet client (or PuTTY in telnet mode)!__ `playd` will
40 do weird things in the presence of Telnet-isms.
41 
42 
43 ## Features
44 
45 * Plays virtually anything [libsox] can play (notably, MP3s, OGGs, and FLACs)
46 * Seek (microseconds, seconds, minutes etc)
47 * Frequently announces the current position
48 * TCP/IP interface with text protocol
49 * Deliberately not much else

52 ## Philosophy
53 
54 `playd` is developed using the following guidelines:
55 
56 * Do one thing and do it well
57 * Be hackable
58 * Favour simplicity over performance
59 * Favour simplicity over features
60 * Let other programs handle the shinies

63 ## Compilation
64 
65 ### Requirements
66 
67 * [libsox] (1.14.1)
68 * [libuv] (0.11.29)
69 * [PortAudio] (19_20140130)
70 * A C++11 compiler (recent versions of [clang], [gcc], and Visual Studio
71  work)
72 
73 Certain operating systems may need additional dependencies; see the OS-specific
74 build instructions below.
75 
76 ### POSIX (GNU/Linux, BSD, OS X)
77 
78 `playd` comes with a GNU-compatible Makefile that can be used both to
79 make and install.
80 
81 To use the Makefile, you'll need [GNU Make] and `pkg-config` (or equivalent),
82 and pkg-config packages for PortAudio, libsox and libuv. We've tested building
83 playd on Gentoo, FreeBSD 10, and OS X, but other POSIX-style operating systems
84 should work.
85 
86 Using the Makefile is straightforward:
87 
88 * Ensure you have the dependencies above;
89 * Read the `Makefile`, to see if any variables need to be overridden for your
90  environment;
91 * Run `make` (or whatever GNU Make is called on your OS; in FreeBSD, for
92  example, it'd be `gmake`), and, optionally, `sudo make install`.
93  The latter will globally install playd and its man page.
94 
95 #### OS X
96 
97 All dependencies are available in [homebrew] - it is highly recommended that
98 you use it!
99 
100 #### FreeBSD (10+)
101 
102 FreeBSD 10 and above come with `clang` 3.3 as standard, which should be able to
103 compile `playd`. `gcc` is available through the FreeBSD Ports Collection
104 and package repositories.
105 
106 You will need `gmake`, as `Makefile` is incompatible with BSD make. Sorry!
107 
108 All of `playd`'s dependencies are available through both the FreeBSD Ports
109 Collection and standard package repository. (The FreeBSD port for PortAudio
110 doesn't build C++ bindings, but we bundle them anyway.) To install them as
111 packages:
112 
113  root@freebsd:/ # pkg install gmake sox libuv portaudio2 pkgconf
114 
115 Then, run `gmake` (__not__ `make`), and, optionally, `gmake install` to install
116 `playd` (as root):
117 
118  user@freebsd:~/ % gmake
119  root@freebsd:~/ # gmake install
120 
121 ### Windows
122 
123 #### Visual Studio
124 
125 _For more information, see `README.VisualStudio.md`._
126 
127 playd **can** be built with Visual Studio (tested with 2013 Premium), but
128 you will need to source and configure the dependencies manually. A Visual
129 Studio project is provided, but will need tweaking for your environment.
130 
131 #### MinGW
132 
133 We haven't managed ourselves, but assuming you can build all the dependencies,
134 (libsox is the difficult one), it should work fine.
135 
136 ### PortAudio C++ Bindings
137 
138 If you have the PortAudio C++ bindings available, those may be used in place of
139 the bundled bindings. This will happen automatically when using the Makefile,
140 if the C++ bindings are installed as a pkg-config package.
141 
142 __Visual Studio users:__ The Visual Studio 7.1 project supplied in the
143 PortAudio source distribution for building the C++ bindings
144 (`\bindings\cpp\build\vc7_1\static_library.vcproj`) should work. If not, then
145 use the bundled bindings.
